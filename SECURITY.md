# Security Policy

## Supported versions
This project is pre-release. Security fixes are applied to the main branch only until a stable release is published.

## Reporting a vulnerability
Please report security issues privately.
- Email: security@example.com
- Do not open public issues for security reports.

Include the following in your report:
- A detailed description of the issue
- Steps to reproduce
- Impact assessment
- Proof of concept (if available)

## Security practices
- Dependency and container scans in CI
- OWASP Top 10 checks for web endpoints
- Secrets are not committed to the repo
